Building & Shipping

ShipRelease something live and usable. "I shipped it" means it's out and people can use it.
MVP — Minimum Viable ProductThe scrappiest version that actually works. Build this, not the dream version.
DeployPut your app live on the internet so others can reach it.
RepoRepository. The folder where all your code lives, usually on GitHub.
CommitSave a version of your code to the repo.
PushSend your committed changes up to the repo online.
PullGrab the latest version of the code down to your machine.
MergeCombine two branches of code into one.
BranchA separate copy of the code where you build a feature without breaking the main version.
Main / MasterThe primary branch. The real, live version of the code.
PR — Pull RequestAsking to merge your branch into main. Others review it before it goes in.
ForkCopy someone else's repo to build on top of it.
CloneDownload a full copy of a repo to work on locally.
StackThe set of tools and tech you built something with. "What's your stack?"
FrontendWhat users see and touch: the buttons, the screens, the design.
BackendThe engine behind the scenes: data, logic, servers.
Full stackSomeone who builds both frontend and backend.
FrameworkA pre-built skeleton that saves you writing everything from scratch (React, Next.js).
LibraryA package of pre-written code you plug in to do a job.
BoilerplateStarter code you reuse to skip the boring setup.
API — Application Programming InterfaceThe socket that lets two apps talk to each other. "Bring your own API key."
EndpointA specific address on an API you send requests to.
DatabaseWhere your app stores its data so it remembers things.
ServerA computer that runs your app and serves it to users over the internet.
CloudSomeone else's servers you rent instead of owning hardware (AWS, Vercel, etc).
LocalRunning on your own machine, not live to the world yet. "Works on my machine."
Prod / ProductionThe live version real users touch. "Don't test in prod."
StagingA copy of prod for testing before you go live.
RollbackUndo a deploy and go back to the last working version.
HotfixAn emergency fix pushed straight to live.
Ship-blockedSomething is stopping you launching. Name it, kill it.
Open sourceCode anyone can view, use, and build on for free.
Legacy codeOld code you inherited that everyone's scared to touch.
CI/CDContinuous Integration / Continuous Deployment. Code gets tested and shipped automatically on every push.
DockerPackages your app and its dependencies into a container so it runs the same everywhere.
ContainerA lightweight box that runs your app isolated from everything else on the server.
K8s — KubernetesSoftware that manages lots of containers across servers. Overkill until it isn't.
MicroservicesSplitting your app into small independent services instead of one big monolith.
MonolithOne big codebase doing everything. Simple to start, painful to scale.
RESTA standard way APIs talk using HTTP methods: GET, POST, PUT, DELETE.
GraphQLAn API style where the client asks for exactly the data fields it needs.
WebhookYour app gets pinged automatically when something happens elsewhere (a payment, a new signup).
Cron jobA scheduled task that runs on a timer. "Every night at 2am, clean up old data."
Env varsEnvironment variables. Secret config like API keys stored outside your code.
MiddlewareCode that sits between a request and your app logic — auth checks, logging, etc.
LinterA tool that catches sloppy code and style issues before you ship.
Package managerInstalls and manages code libraries (npm, pip, cargo).
DependencyExternal code your project relies on. Each one is a potential breaking point.
SemVerSemantic versioning. Major.Minor.Patch (2.1.0). Breaking changes bump the first number.
DevOpsThe practice of developers owning deployment and infrastructure, not throwing code over a wall.
SRE — Site Reliability EngineeringMaking sure your app stays up, fast, and recoverable when things break.
CDNContent Delivery Network. Serves your files from servers close to users so pages load fast.
DNSThe phone book of the internet. Turns domain names into server IP addresses.
SSL / TLSEncryption for websites. The padlock in your browser. HTTPS depends on it.
Load balancerDistributes traffic across multiple servers so one doesn't get crushed.
CacheStoring frequently used data in fast memory so you don't re-fetch it every time.
RedisA super-fast in-memory database used for caching, sessions, and queues.
QueueA line of jobs waiting to be processed. Send emails later, not during the user's click.
LatencyDelay. How long between a request and a response. Lower is better.
UptimePercentage of time your app is online. 99.9% sounds great until you do the math.
ServerlessRun code without managing servers. You pay per execution, not per idle machine.
WebSocketA persistent two-way connection. Powers real-time chat, live updates, multiplayer.
CRUDCreate, Read, Update, Delete. The four basic operations on data.
ORMObject-Relational Mapping. Talk to your database using code objects instead of raw SQL.
NoSQLNon-relational databases (MongoDB, DynamoDB). Flexible schemas, different tradeoffs.
MigrationA versioned change to your database schema. Adds columns, tables, indexes.
Feature flagToggle features on/off without deploying. Ship code dark, flip it on when ready.
A/B testShow two versions to different users and measure which performs better.
Canary deployRoll out to a tiny slice of users first. If it breaks, only a few people notice.
MonorepoMultiple projects in one repo. Google does it. Your team might hate it.
SDKSoftware Development Kit. Pre-built tools to integrate with a platform faster.
CLICommand Line Interface. Control tools by typing commands in a terminal.
IDEIntegrated Development Environment. VS Code, Cursor, JetBrains — where you write code.
ObservabilityLogs, metrics, and traces that tell you what's actually happening in production.
IncidentWhen prod is broken and users are affected. All hands on deck.
PostmortemA blameless write-up after an incident. What broke, why, and how to prevent it.
SLAService Level Agreement. A promise of uptime or response time, often with penalties.
RebaseReplay your commits on top of the latest main branch. Cleaner history, more risk.
Merge conflictTwo people changed the same lines. Git can't auto-merge. You fix it manually.
🤖02

AI & Agents

AI — Artificial IntelligenceSoftware that performs tasks that normally need human intelligence.
LLM — Large Language ModelThe AI brain behind chatbots: Claude, GPT, Gemini. Trained on huge amounts of text.
ML — Machine LearningTeaching software to learn patterns from data instead of hard-coding rules.
ModelThe specific AI you're using. Different models, different strengths and costs.
PromptWhat you type to tell the AI what to do.
Prompt engineeringThe skill of asking the AI in a way that gets great results.
System promptThe hidden instructions that set the AI's behavior before you even type.
TokenThe unit AI reads and charges by. Roughly a word-chunk. More tokens, more cost.
Context windowHow much the AI can hold in mind at once. Fill it up and it forgets the start.
AgentAI that takes actions on its own, not just chats. It can code, browse, and execute.
AgenticDescribes AI that acts with autonomy, making decisions and taking steps toward a goal.
Agentic workflowA chain of steps an AI agent runs mostly on its own: plan, act, check, repeat, until the job's done.
Agentic codingUsing an AI agent to write, run, and fix code for you (Claude Code, Cursor).
Fine-tuneTraining a model on your own data to specialize it for your use case.
RAG — Retrieval Augmented GenerationFeeding the AI your documents so it answers from them, not just its training.
EmbeddingTurning text into numbers so AI can search and compare meaning.
Vector databaseWhere embeddings live. Powers AI search and memory.
HallucinationWhen the AI confidently makes something up. Always check.
TemperatureHow random or creative the AI's output is. Low = focused, high = wild.
InferenceThe AI actually running and giving you an answer.
MultimodalAn AI that handles text, images, audio, and video, not just words.
Open source modelA model you can download and run yourself, often free (Llama, Mistral).
Local modelAn AI running on your own hardware. No API costs, full privacy.
API keyYour personal access pass to use an AI or service. "Bring your own key."
Rate limitThe cap on how many requests you can make in a window.
Pipeline / ChainLinking multiple AI steps together into one automated workflow.
Few-shotGiving the AI a few examples in your prompt so it copies the pattern.
Zero-shotAsking the AI to do something with no examples at all.
Training dataThe text and content a model learned from.
ParametersThe internal dials a model tunes while learning. More is roughly more capable (and costly).
GPUThe chip that powers AI. The pickaxes of the AI gold rush.
AGI — Artificial General IntelligenceHypothetical AI as capable as a human across any task. Not here yet.
JailbreakTricking an AI past its safety rules. We don't do this here.
MCP — Model Context ProtocolA standard that lets AI models plug into external tools and data sources.
TransformerThe architecture behind modern LLMs. Attention is all you need, literally.
Neural networkLayers of math that learn patterns from data. The building block of deep learning.
Deep learningNeural networks with many layers. Powers image recognition, language models, and more.
NLPNatural Language Processing. Teaching computers to understand and generate human language.
RLHFReinforcement Learning from Human Feedback. How chatbots learn to be helpful and safe.
Function callingWhen an AI decides to call a tool or API instead of just returning text.
GuardrailsSafety rules that stop AI from doing harmful or off-topic things.
QuantizationShrinking a model's precision to run faster and cheaper with minor quality loss.
Synthetic dataAI-generated training data. Useful when real data is scarce or sensitive.
OverfittingModel memorized training data but fails on new inputs. Classic ML mistake.
BenchmarkStandard tests to compare model performance. MMLU, HumanEval, etc.
Inference costWhat you pay every time the model runs. Tokens × price adds up fast.
StreamingAI responses arriving word by word in real time instead of all at once.
Structured outputForcing AI to return JSON or a schema instead of free-form text.
Computer visionAI that understands images and video — object detection, OCR, face recognition.
CopilotAI embedded in your workflow suggesting code, text, or actions as you work.
AI wrapperA thin product built on top of someone else's model API. Easy to ship, hard to defend.
Model collapseWhen AI trained on AI-generated content gets worse over generations.
Prompt injectionTricking an AI by hiding instructions in user input. A real security problem.
⛓️03

Crypto & Web3

BlockchainA shared digital ledger no single person controls. Records are chained together and can't be quietly changed.
CryptoDigital money that runs on a blockchain instead of a bank.
WalletWhere you store crypto. Holds your keys, not literally the coins.
Private keyThe secret password that controls your wallet. Lose it, lose everything. Never share it.
Seed phrase12 or 24 words that back up your wallet. Guard them like your life.
Web3The idea of an internet built on blockchains, owned by users not corporations.
Smart contractCode on a blockchain that runs automatically when conditions are met. No middleman.
GasThe fee you pay to do anything on a blockchain.
On-chainAnything recorded directly on the blockchain, public and permanent.
TokenA crypto asset built on top of an existing blockchain (in crypto context, not AI).
MemecoinA crypto token based on a joke or trend. High risk, high volatility, mostly a casino.
NFT — Non-Fungible TokenA one-of-a-kind digital item recorded on a blockchain.
DeFi — Decentralized FinanceFinancial tools (lending, trading) run by code on a blockchain, no bank.
DEX — Decentralized ExchangeA crypto exchange with no company in the middle, just smart contracts.
DAO — Decentralized Autonomous OrganizationA group run by code and member votes instead of bosses.
SolanaA fast, cheap blockchain popular for trading and memecoins.
EthereumThe blockchain that pioneered smart contracts and most of Web3.
StablecoinA crypto token pegged to a stable value like the dollar.
Rug / Rug pullWhen a project's creators take the money and vanish. A scam.
Ape / Aping inBuying into something fast without much research.
Diamond hands / Paper handsHolding through the fear vs selling at the first scare.
DYORDo Your Own Research. The crypto disclaimer for everything.
FUDFear, Uncertainty, Doubt. Negative talk, real or manufactured.
WhaleSomeone holding a huge amount of an asset, big enough to move the market.

Dev Slang

Vibe codingBuilding by feel with AI, describing what you want and letting it write the code.
Ship itStop polishing, release it now. The Ora religion.
Yak shavingDoing endless prep tasks instead of the actual thing. The enemy.
BikesheddingArguing over tiny details while the big thing rots.
Rubber duckingExplaining your problem out loud until you spot the answer yourself.
DogfoodingUsing your own product to prove it works.
Foot-gunA feature that lets you blow your own foot off. Easy to misuse.
Spaghetti codeMessy code tangled beyond saving.
Tech debtShortcuts you take today that you pay for later.
HackyA solution that works but is ugly and fragile.
JankyBuggy, unstable, held together with tape.
Nuke itDelete the whole thing and start fresh.
RefactorRewrite code to be cleaner without changing what it does.
GreenfieldA brand new project with no old code to deal with.
BrownfieldWorking inside an existing, messy, established codebase.
Happy pathThe ideal flow where everything works. Test the unhappy paths too.
Edge caseThe weird rare situation that breaks everything.
BugSomething broken in your code.
Feature, not a bugThe joke you make when your bug turns out useful.
Works on my machineThe classic excuse when it breaks for everyone else.
LGTMLooks Good To Me. Approval on a review.
WIPWork In Progress.
TL;DRToo Long; Didn't Read. The short version.
RTFMRead The Manual. Politely.
+1I agree, me too.
Cook / CookingBuilding something great. "Let him cook."
CrackedInsanely skilled. "That dev is cracked."
BasedRespectably bold, true to yourself.
GoatedThe greatest of all time.
Lock inGo heads-down and focus hard.
Send itCommit fully, launch it, go.
Shipping fastReleasing often instead of waiting for perfect.
10x engineerMythical dev who outputs ten times more. Usually just means "really good and focused."
Glue codeCode that connects systems together. Not glamorous, always necessary.
Cargo cultingCopying patterns from big tech without understanding why they exist.
Premature optimizationMaking code fast before you know if anyone will use it. Root of all evil.
Not invented hereRefusing to use existing tools because "we can build it better." Usually wrong.
Scope creepFeatures keep getting added until the project never ships.
Analysis paralysisOverthinking so long you never start building.
Bleeding edgeSo new the docs don't exist yet. Fun until prod breaks at 3am.
Bus factorHow many people could get hit by a bus before the project is doomed. Higher is better.
Rubber stampApproving a PR without actually reading it. Happens more than anyone admits.
P0 / P1Priority levels. P0 = drop everything, prod is on fire. P1 = important, not existential.
Sev-1Severity 1 incident. Critical outage affecting users right now.
On-callYou're the one who gets paged when prod breaks at midnight.
PagerDutyThe app that wakes you up when something's down. Every engineer's nemesis.

Business & Money

MRR — Monthly Recurring RevenueThe number that matters. Price times members.
ARR — Annual Recurring RevenueMRR times twelve.
ChurnPeople who cancel. The silent killer.
RetentionPeople who stay. The opposite of churn, the real game.
CAC — Customer Acquisition CostWhat it costs to get one buyer.
LTV — Lifetime ValueWhat a member is worth over their whole time with you.
RunwayHow long your money lasts before you're broke.
Burn rateHow fast you spend money each month.
PMF — Product-Market FitThe moment people want it faster than you can ship. Chase this.
BootstrapBuild with your own money, no investors.
Pre-seed / SeedThe earliest funding rounds if you raise money.
Cap tableWho owns what percentage of your company.
MoatThe thing that stops competitors copying you.
FlywheelA loop where each part feeds the next and it compounds over time.
FunnelThe path from stranger to paying customer.
Top of funnelWhere new people first find you.
Conversion rateThe percentage who take the action you want (join, buy).
LeadA potential customer who's shown interest.
Warm leadSomeone already interested. Easier to convert.
Cold leadSomeone who's never heard of you.
UpsellGetting an existing customer onto a higher tier.
FreemiumFree tier as the front door, paid tier as the product.
North Star metricThe single number your whole team optimizes for.
Ramen profitableEarning just enough to cover your basic living. The first real milestone.
TractionProof it's working: growth, revenue, users.
PivotChanging your core idea when the first one isn't landing.
IterateImprove in small fast loops based on real feedback.
SaaSSoftware as a Service. Pay monthly for software hosted in the cloud. The default startup model.
B2BBusiness to Business. You sell to companies, not consumers.
B2CBusiness to Consumer. You sell directly to everyday people.
PLG — Product-Led GrowthThe product itself drives signups and upgrades. Free tier → paid, no sales team needed.
GTM — Go-To-MarketYour plan for how you reach customers and make money. Strategy, not hope.
ICP — Ideal Customer ProfileThe exact type of customer your product is built for. Build for them, not everyone.
TAMTotal Addressable Market. The entire pie if you owned 100% of a market.
SAMServiceable Addressable Market. The slice of TAM you can realistically reach.
SOMServiceable Obtainable Market. What you can actually capture in the near term.
Unit economicsProfit or loss per customer. If CAC > LTV, you lose money on every sale.
ARPUAverage Revenue Per User. Total revenue divided by number of users.
NRR — Net Revenue RetentionRevenue from existing customers over time. Above 100% means expansion beats churn.
DAU / MAUDaily / Monthly Active Users. How many people actually use your product.
ActivationThe moment a new user gets real value from your product. The metric that predicts retention.
OnboardingThe flow that gets new users to their first win. Bad onboarding = instant churn.
Network effectsProduct gets better as more people use it. Social networks, marketplaces, dev tools.
MarketplacePlatform connecting buyers and sellers. Uber, Airbnb, Etsy. Hard to bootstrap.
Two-sided marketYou need both supply and demand. Chicken-and-egg problem until you crack it.
UnicornA startup valued at $1B+. Rare. Most "unicorns" never actually get there.
ExitHow founders and investors cash out — acquisition or IPO.
AcquisitionA bigger company buys your startup. The most common "successful" outcome.
IPOInitial Public Offering. Your company sells shares on the stock market. The big leagues.
Acqui-hireBuying a company mainly for the team, not the product. Common in tech.
EquityOwnership in the company. Usually given to founders, employees, and investors.
Stock optionsThe right to buy company shares at a set price later. Standard startup compensation.
VestingEarning your equity over time. Usually 4 years with a 1-year cliff.
CliffPeriod before any equity vests. Leave before the cliff, get nothing.
DilutionYour ownership percentage shrinks when new shares are issued to investors.
Option poolShares reserved for future employees. Comes out of founders' ownership.
Pitch deckSlides that tell your startup story to investors. Problem, solution, traction, ask.
Elevator pitchYour startup explained in 30 seconds. If you can't do this, you're not ready.
Demo dayAccelerators' event where startups pitch to a room full of investors.
AcceleratorProgram that gives startups money, mentorship, and connections (YC, Techstars).
IncubatorEarly-stage support without the intense batch structure of an accelerator.
YC — Y CombinatorThe most famous startup accelerator. "Make something people want."
Product HuntWhere startups launch new products to the tech community. Launch day matters.
Indie hackerSolo builder shipping profitable products without VC. Bootstrap mindset.
Default aliveYour revenue covers costs without raising more money. The goal for bootstrappers.
Default deadYou need to raise or cut costs to survive. Paul Graham's framing.
Zombie startupAlive but not growing. Neither failing nor succeeding. Stuck in limbo.
Land and expandStart small with a customer, then grow the deal over time. Classic B2B SaaS motion.
ACV — Annual Contract ValueAverage yearly revenue per customer contract. Key for enterprise sales.
MQL / SQLMarketing / Sales Qualified Lead. How far along someone is in the buying journey.
PQL — Product Qualified LeadUser who's shown buying intent through product usage, not a form fill.
Founder-led salesFounders sell the product themselves before hiring a sales team. You should do this first.
Usage-based pricingCharge based on how much customers use. API calls, storage, seats.
Free trialLet users try before they pay. Converts better than demos for self-serve products.
NPS — Net Promoter Score"Would you recommend us?" Scored -100 to 100. Vanity metric if you stop there.
Customer successTeam that makes sure paying customers get value and don't churn.
OKRObjectives and Key Results. Goal-setting framework. Objectives are what, key results are how you measure.
RoadmapPlanned features and timeline. Share carefully — customers treat it as a promise.
POC — Proof of ConceptQuick build to prove an idea works before committing real resources.
BetaEarly version shared with select users before public launch. Expect bugs.
GA — General AvailabilityOfficial public launch. No longer beta, no longer "coming soon."
WaitlistCollect emails before launch. Builds hype and validates demand.
Cohort analysisTracking groups of users over time to see if retention is improving.
Rule of 40Growth rate + profit margin should exceed 40%. SaaS health check.
Burn multipleHow much you burn to generate $1 of new ARR. Lower is more efficient.
Payback periodMonths to recover CAC from a customer's revenue. Shorter = healthier.
Magic numberSales efficiency metric. New ARR divided by sales & marketing spend last quarter.
BlitzscalingPrioritizing speed over efficiency to win a market. High risk, high reward.
Category creationInventing a new market category instead of competing in an existing one.
Switching costsPain of leaving your product. High switching costs = lower churn.
Platform riskBuilding on someone else's platform (Apple, Google, Twitter). They can kill you overnight.
StripeThe default payment processor for startups. "Payments for the internet."

VC & Fundraising

VC — Venture CapitalMoney from funds that invest in high-growth startups in exchange for equity.
Angel investorWealthy individual who invests their own money in early startups.
Term sheetThe document outlining investment terms before lawyers draft the real paperwork.
SAFESimple Agreement for Future Equity. Invest now, get shares later at a discount or cap.
Convertible noteA loan that converts to equity in the next funding round. Older alternative to SAFEs.
Pre-money valuationWhat your company is worth before the investment goes in.
Post-money valuationWhat your company is worth after the investment. Pre-money + investment amount.
Valuation capMaximum valuation at which a SAFE converts. Protects early investors.
Discount ratePercentage discount SAFE investors get when their money converts to shares.
Series A / B / CNamed funding rounds as startups grow. Each round is usually bigger and at a higher valuation.
Down roundRaising at a lower valuation than last time. Painful signal to the market.
Up roundRaising at a higher valuation. The good kind.
Bridge roundShort-term funding to survive until the next proper round.
Extension roundMore money from existing investors in the same round. Buys time.
Party roundLots of small investors, no clear lead. Can signal weak conviction.
Lead investorThe VC who puts in the most money and sets the terms. Others follow them.
Follow-onWhen an existing investor puts more money into a later round.
Pro rataRight to invest in future rounds to maintain your ownership percentage.
Due diligenceInvestors digging into your finances, code, legal, and team before writing a check.
Data roomSecure folder of documents investors review during due diligence.
Liquidation preferenceInvestors get paid first if the company sells. 1x means they get their money back first.
Preferred stockInvestor shares with special rights. Founders usually hold common stock.
Board seatA spot on the company's board of directors. Lead investors usually get one.
Board of directorsGroup that oversees major company decisions. Founders, investors, independents.
Protective provisionsInvestor veto rights on big decisions like selling the company or raising more money.
Drag-alongForces minority shareholders to sell if the majority agrees to an acquisition.
Tag-alongLets minority shareholders sell on the same terms if majority sells.
Fully dilutedOwnership calculated including all options, SAFEs, and convertible notes.
GP / LPGeneral Partner runs the fund. Limited Partners put in the money (pensions, endowments).
Fund sizeTotal capital a VC firm has raised to invest. A $100M fund makes very different bets than $1B.
Check sizeHow much a VC typically invests per deal. Match your stage to their check size.
PortfolioAll the companies a VC fund has invested in.
CarryThe GP's share of fund profits, usually 20%. How VCs actually get rich.
2 and 20Standard VC fee structure. 2% annual management fee, 20% carry on profits.
TVPITotal Value to Paid-In. Fund performance metric. Above 1x means the fund is up.
IRRInternal Rate of Return. Annualized return on investment. VCs live by this number.
MOICMultiple on Invested Capital. 3x means they tripled their money.
Corporate VCInvestment arm of a big company (Google Ventures, Salesforce Ventures).
Family officeManages wealth for a rich family. Can invest as angels or through funds.
SyndicateGroup of angels pooling money to invest together in one deal.
SPVSpecial Purpose Vehicle. A one-deal fund to pool investors into a single startup.
Rolling fundVC fund that raises quarterly instead of every 10 years. More flexible, newer model.
ScoutSomeone who finds deals for a VC fund in exchange for a small carry share.
Warm introGetting introduced to a VC through someone they trust. Cold emails barely work.
Partner meetingThe meeting where all VC partners decide yes or no on your deal.
PassWhen a VC decides not to invest. You'll hear "no" hundreds of times.
Deal memoInternal VC document arguing why they should invest in you.
Investment thesisA VC's stated focus — stage, sector, geography. Pitch the ones that match.
No-shop clauseYou can't talk to other investors for a set period while negotiating terms.
ExclusivitySame idea — one investor gets exclusive negotiating rights for a window.
83(b) electionUS tax filing when you get founder shares. File within 30 days or pay way more tax later.
QSBSQualified Small Business Stock. US tax benefit that can exclude millions in capital gains.
Delaware C-CorpThe default US incorporation for VC-backed startups. Investors expect it.
Cap table managementTracking who owns what. Tools like Carta and Pulley handle this.
Founder vestingInvestors make founders earn their shares too. Protects against a founder leaving early with half the company.
Single-trigger accelerationAll your unvested shares vest instantly if the company is acquired.
Double-trigger accelerationShares vest on acquisition only if you're also fired or demoted.
Anti-dilutionProtects investors if you raise at a lower valuation. Can really hurt founders.
Revenue multipleValuation divided by ARR. SaaS companies often valued at 10-20x ARR.
CompsComparable companies used to justify your valuation. "We're like X but for Y."
MarkupWhen a VC increases the paper value of your company in their portfolio.
Write-downWhen a VC admits an investment is worth less than they paid.
Vintage yearThe year a VC fund started investing. Performance is compared by vintage.
Deploy paceHow fast a VC fund invests its capital. Slow deploy = picky or struggling to find deals.
ReservesMoney a VC fund holds back to invest more in their best portfolio companies.
EIREntrepreneur in Residence. Experienced founder hanging at a VC firm before their next startup.
Solo GPOne person running an entire VC fund. More common now, less bureaucracy.
Emerging managerA VC raising their first or second fund. Hungrier, but less brand power.
Tier 1 VCTop firms like Sequoia, a16z, Accel. Brand name on your cap table opens doors.
Signal riskWhen a well-known VC passing on your round scares off other investors.
Inside roundOnly existing investors participate. Can signal no new interest from outside.
Priced roundInvestment at a set valuation with preferred shares. More formal than a SAFE.
Uncapped SAFESAFE with no valuation cap. Investor converts at whatever the next round's price is.
Side letterSpecial terms given to one investor that aren't in the main deal documents.
MFNMost Favored Nation. SAFE clause giving you the best terms any other early investor got.
Capital callWhen a fund asks LPs to actually send the money they committed.
DPIDistributions to Paid-In. How much cash LPs have actually received back from the fund.
🔒07

Security

CybersecurityProtecting systems, networks, and data from digital attacks.
AppSecApplication Security. Finding and fixing vulnerabilities in your code.
DevSecOpsBuilding security into the development pipeline, not bolting it on at the end.
Threat modelMapping out what you're protecting, from whom, and how they might attack.
Attack surfaceEvery entry point an attacker could exploit. Smaller surface = fewer headaches.
VulnerabilityA weakness in your system that could be exploited. Patch them fast.
CVECommon Vulnerabilities and Exposures. Public catalog of known security flaws.
Zero-dayA vulnerability nobody knows about yet — no patch exists. The scariest kind.
Pen testPenetration test. Ethical hackers try to break into your system and report what they find.
Bug bountyPaying security researchers who find and report vulnerabilities responsibly.
OWASPOpen Web Application Security Project. Publishes the top 10 web app vulnerabilities every dev should know.
XSSCross-Site Scripting. Attacker injects malicious scripts into your web pages.
CSRFCross-Site Request Forgery. Tricks a logged-in user into doing something they didn't intend.
SQL injectionAttacker manipulates your database through unsanitized input. Classic and still common.
SSRFServer-Side Request Forgery. Your server is tricked into making requests to internal systems.
RCERemote Code Execution. Attacker runs arbitrary code on your server. Game over.
IDORInsecure Direct Object Reference. Accessing someone else's data by changing an ID in the URL.
Credential stuffingUsing leaked username/password combos from other breaches to break into your app.
Brute forceTrying every possible password until one works. Rate limiting stops this.
MFA / 2FAMulti-Factor / Two-Factor Authentication. Password plus a code from your phone.
SSOSingle Sign-On. Log in once, access multiple apps. "Sign in with Google."
OIDCOpenID Connect. Standard protocol behind modern SSO and social login.
JWTJSON Web Token. A signed token that proves who you are. Don't store secrets in them.
Session hijackingStealing someone's active session to impersonate them.
API key leakExposing a secret key in public code or logs. Bots scan GitHub for these 24/7.
Secrets managerSecure vault for API keys and passwords. Vault, AWS Secrets Manager, Doppler.
Hardcoded credentialsPutting passwords directly in source code. Never do this. Ever.

bcrypt / Argon2Proper password hashing algorithms. Use these, not MD5 or SHA-1.

PCI DSSPayment card security standard. Required if you handle credit card data directly.
ComplianceMeeting legal and regulatory security requirements for your industry.

Supply chain attackCompromising a dependency or vendor to reach your users. SolarWinds-style attacks.

Shift leftMoving security checks earlier in development. Cheaper to fix bugs before production.
SBOMSoftware Bill of Materials. A list of every dependency in your app and its version.

Social engineeringManipulating people instead of systems. "Hey, I'm from IT, what's your password?"
RansomwareMalware that encrypts your data and demands payment to unlock it.

CORSCross-Origin Resource Sharing. Browser rules for which domains can call your API.
CSPContent Security Policy. HTTP header that blocks unauthorized scripts from running.
Input validationChecking user input before processing it. Your first line of defense.
SanitizationCleaning user input to remove dangerous characters before display or storage.

Fail closedWhen security checks fail, deny access. Don't fail open and let everyone in.
MisconfigurationThe #1 cloud security issue. Public S3 buckets, open databases, default passwords.
Container securityScanning Docker images for vulnerabilities and running them with minimal permissions.

CVSSCommon Vulnerability Scoring System. Rates severity of CVEs from 0 to 10.
